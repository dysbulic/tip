File Access in Templ

The Templ CMS uses a two-stage document generation process. The first bit is a modular java-based hooks system. This generates the actual web content, be it php, html, svg or whatever. When doing the preprocessing, the system needs to regenerate smallest subset of files possible or the system won't scale.

Content for template combination comes from metadata. Metadata is a set of named lists of DOM nodes. Take this example:

File: default.html.tmpl

<html xmlns="http://www.w3.org/1999/xhtml"
      xmlns:tmpl="http://odin.himinbi.org/templ/0.1/"
      xmlns:xi="http://www.w3.org/2001/XInclude">
  <head><title><tmpl:var name="title"/></title></head>
  <body>
    <tmpl:var name="title"/>
    <div id="sidebar">
      <lst:listing src="../news"/>
    </div>
  </body>
</html>

This file is loaded according to the default load order.
