Templ Default Settings

In designing the Templ CMS, I had three main thoughts going through my mind that I wanted to address:

  • I'm a web designer. I am comfortable with PHP, javascript, XHTML and XSLT. Other competent designers will be as well and so the underlying page structure can use these elements without a penalty to readability or maintainability.
  • I do not want to maintain all the content myself. The people maintaining the content are not web designers. They need something WYSIWYG.
  • At some point someone will ask me to do something I never thought of and which will require a different paradigm than I've structured.

My goal with the system was to have it do something reasonable that would work correctly 90% of the time. Also, to leave things open to configuration at various levels to the extent that whatever changes are needed to fix that last 10% can be done.

The core of the system is HooksProcessor. If this class is invoked with a directory as an argument, it will iterate over each of the files in the directory. For all the files that match load a set of default hooks.

The first of these hooks is org.himinbi.templ.LoadMetadataHook. It takes <tmpl:var name="path"> arguments for pathes to load, or if those are missing it loads a default set from the following places:

  • ~/.templ.metadata
  • .templ.metadata between the original directory and the current directory (directories are processed recursively)
