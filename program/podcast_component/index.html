Podcast Joomla Component

I have looked at a couple different CMSes for the new Indiefeed site. I am finding Drupal a bit cumbersome to deal with. Joomla is generally respected as a cleaner product, so I am going to attempt to develop the site as a custom component.

Information Architecture

The first step is to describe the information that needs to be presented:

The site is for displaying podcasts. Each podcast has:

  • Artist
  • Track Title
  • Artist Graphic
  • Show URL
  • List of Buy Links
  • Show Description
  • Ranking
  • Comments

There are several data files and a program to load the data.

  • indiefeed_altrock.xml
  • indiefeed_blues.xml
  • indiefeed_electronica.xml
  • indiefeed_indie.xml
  • indiefeed_bigshed.xml
  • indiefeed_poetry.xml
  • indiefeed_hiphop.xml
  • Frontpage
  • Archives

Installation instructions including setting the podcast component to the default.

The component is decomposed into several files:

  • podcast.php • main redirector that routes incoming task requests
  • podcast.class.php • php class encapsulating the database access logic
  • podcast.html.php • handles the actual rendering of podcasts, primarily redirects to functions defined in:
    • podcast.listing.php • generates the frontpage listing
    • podcast.archive.php • generates the channel archive
    • podcast.detail.php • displays information about a specific podcast
  • podcast.edit.php
